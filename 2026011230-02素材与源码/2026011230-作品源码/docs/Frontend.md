# 前端说明（Vue 3 + TypeScript）

## 运行与构建
- 开发服务器：进入 frontend，执行 `npm run dev`，浏览器打开 http://localhost:5173
- 构建产物：`npm run build`，输出在 dist/ 目录，适合静态托管。

## 架构设计
- 组件化设计：基于 Vue 3 的组合式 API，使用 script setup。
- 路由与状态：vue-router + Pinia，路由守卫处理认证跳转，状态在 Pinia store 中管理。
- UI 框架：Element Plus，图标库采用 @element-plus/icons-vue。
- 与后端的对接：通过 axios 请求后端 REST API，集中在 services 层处理客户端 API 调用。
- 国际化与可访问性：尽量使用 i18n 支持与 A11y 友好模式。
- 性能与响应性：按需加载路由组件、异步组件、避免不必要的计算、懒加载图片等。
- 主题与样式：使用 tailwind 风格的类与自定义 CSS，确保响应式布局。

## 运行时风格与最佳实践
- 状态管理：尽量复用状态，避免无必要的重复数据查询。
- API 错误处理：在 UI 中提供统一的错误提示、加载指示与兜底逻辑。
- 表单与验证：使用自定义验证逻辑，保持 UX 一致性。
- 路由与权限：保护需要鉴权的路由，未授权跳转到登录页。
- 响应式与无障碍：确保在移动端的可用性与基本可访问性。

## UI/组件设计
- 常用布局：Card、Table、Form、Modal、Tabs 等组件按需组合。
- 组件通信：Props/Events、Provide/Inject、Pinia store 进行状态共享。
- 可复用性：抽取通用组件，减少重复代码。

## 性能优化要点
- 使用懒加载路由组件实现初始加载最小化。
- 对频繁渲染的组件用异步数据，以及合理的 loading 状态。
- 图片与资源的懒加载、缓存策略以及资源压缩。

## 国际化与本地化
- 统一的文本资源文件，按区域分目录管理（如 en-US、zh-CN）。
- 日期、数字等本地化格式通过国际化库处理。

## 测试与质量保障
- 单元测试：组件与工具函数，使用 Vue Testing Library 与 Vitest。
- 集成测试：路由、页面与 API 的端到端场景（可结合 Cypress）。
- 代码风格：ESLint、Prettier、TypeScript 配置保持一致。
- CI/CD：在 PR/提交时触发测试与构建。

## 安全与可访问性
- 表单校验、输入限制、错误提示要清晰友好。
- 键盘导航、屏幕阅读器友好、对比度等无障碍要点。

## 与后端协作示例
- API 调用路径、参数、返回类型以 docs/API.md 为准。
- 当后端接口变更时，请同步更新类型定义和调用逻辑。
