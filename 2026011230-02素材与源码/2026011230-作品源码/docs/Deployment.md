# 部署指南

本节提供本地与基本生产部署的参考方案，优先使用 Docker Compose 进行本地开发与测试。生产环境可根据实际云端平台进行适配。

版本前提
- 该方案假设后端使用 Spring Boot、前端为 Vue 3 + TS，数据库为 MySQL，缓存 Redis，消息队列 RabbitMQ。

本地开发（Docker Compose）
版本: 3.8

services:
- backend:
  说明：后端服务，8080 暴露端口，依赖 db/redis/rabbitmq。
- frontend:
  说明：前端服务，5173 暴露端口，依赖 backend。
- db:
  说明：MySQL 数据库镜像。
- redis:
  说明：Redis 缓存镜像。
- rabbitmq:
  说明：RabbitMQ 镜像，包含管理界面。

部署要点
- 数据库初始化：首次启动后请执行数据库初始化脚本或按后端应用自带的迁移策略完成表结构创建。
- 环境分离：生产环境应使用独立的数据库、缓存与消息队列实例，避免与开发环境混用。
- 端口与域名：将前、后端暴露端口映射为实际访问域名的端口，必要时通过反向代理（Nginx/Traefik）转发。
- 日志与监控：配置日志级别，集成监控（如 Prometheus、Grafana）以观察应用健康状况。

生产建议
- 使用持久化存储、数据库集群/备份、以及 secrets 管理来保护敏感信息。
- 将前端与后端打包后通过静态托管与 API 网关对外暴露。
- 考虑蓝绿/灰度发布策略以降低上线风险。

维护与升级
- 数据库变更应使用版本化的迁移工具（如 Flyway 或 Liquibase）。
- 在升级前务必备份数据与配置。

进一步定制
- 根据实际部署环境调整 JVM 参数、GC 策略与资源限制。
