# 贡献指南
- 运行相关测试用例（前端/后端，视任务而定）
- 确认本次改动不会破坏现有功能。
- 提交信息应包含类型标签（feat、fix、docs、chore、refactor 等）以及简要描述。
- 示例： feat(frontend): add login page routing
- 1) Fork 项目（如使用 GitHub 等托管平台）
- 2) 在本地创建 feature 分支，完成实现
- 3) 提交 PR，并在描述中给出变更原因与影响
- 4) 通过 CI/检查后合并
- 
- 代码审查要点
- 1) 变更的范围与影响点
- 2) 是否新增/修改了公iple API 接口
- 3) 是否增加了测试覆盖
- 4) 安全性与隐私影响
- 5) 文档是否同步更新
- 
- 安全与合规
- 遵循项目的安全最佳实践，避免将密钥、证书等敏感信息提交到代码库。
- 如涉及第三方依赖的安全漏洞，请在 PR 中说明修复方案与回滚计划。
